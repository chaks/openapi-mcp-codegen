# Generated Code Structure

This document describes the structure and content of code generated by OpenAPI MCP Codegen.

## Overview

For a root package of `io.swagger.petstore`, the generator creates the following structure:

```
generated/
├── build.gradle.kts              # Gradle build configuration
├── settings.gradle.kts           # Gradle settings file
├── src/main/kotlin/
│   └── io/swagger/petstore/
│       ├── domain/               # Data classes
│       │   ├── Pet.kt
│       │   ├── Category.kt
│       │   └── Order.kt
│       ├── client/               # REST client interface
│       │   └── PetstoreClient.kt
│       └── tool/                 # MCP tool wrappers
│           └── PetstoreTools.kt
└── src/main/resources/
    └── application.properties     # Quarkus configuration
```

## Package Structure

### Domain Package (`*.domain`)

Contains data classes representing the schema definitions from the OpenAPI specification.

**Features:**
- Kotlin data classes with `@kotlinx.serialization.Serializable`
- Jackson annotations (`@JsonProperty`, `@JsonInclude`, `@JsonPropertyOrder`)
- JSON-B annotations (`@JsonbProperty`) for compatibility
- Proper nullability based on `required` fields
- Nested type support

**Example:**

```kotlin
package io.swagger.petstore.domain

import com.fasterxml.jackson.annotation.JsonInclude
import com.fasterxml.jackson.annotation.JsonProperty
import com.fasterxml.jackson.annotation.JsonPropertyOrder
import jakarta.json.bind.annotation.JsonbProperty

/**
 * A pet for sale in the pet store
 */
@kotlinx.serialization.Serializable
data class Pet(
    @JsonProperty("id")
    @JsonbProperty("id")
    val id: Long? = null,

    @JsonProperty("name")
    @JsonbProperty("name")
    val name: String,

    @JsonProperty("category")
    @JsonbProperty("category")
    val category: Category? = null,

    @JsonProperty("status")
    @JsonbProperty("status")
    val status: String? = null
)
```

### Client Package (`*.client`)

Contains REST client interfaces using MicroProfile Rest Client.

**Features:**
- `@RegisterRestClient` annotation for Quarkus integration
- JAX-RS annotations (`@GET`, `@POST`, `@PUT`, `@DELETE`, `@PATCH`)
- Parameter annotations (`@PathParam`, `@QueryParam`, `@HeaderParam`, `@CookieParam`)
- Request body handling with `@Consumes`
- Response type declarations

**Example:**

```kotlin
package io.swagger.petstore.client

import jakarta.ws.rs.Consumes
import jakarta.ws.rs.DELETE
import jakarta.ws.rs.GET
import jakarta.ws.rs.POST
import jakarta.ws.rs.PUT
import jakarta.ws.rs.Path
import jakarta.ws.rs.PathParam
import jakarta.ws.rs.Produces
import jakarta.ws.rs.QueryParam
import jakarta.ws.rs.core.MediaType
import org.eclipse.microprofile.rest.client.annotation.RegisterRestClient

@RegisterRestClient(configKey = "petstore-api")
interface PetstoreClient {

    @GET("/pet/findByStatus")
    fun findPetsByStatus(
        @QueryParam("status") status: List<String>
    ): List<Pet>

    @GET("/pet/{petId}")
    fun getPetById(
        @PathParam("petId") petId: Long
    ): Pet

    @POST("/pet")
    @Consumes(MediaType.APPLICATION_JSON)
    @Produces(MediaType.APPLICATION_JSON)
    fun addPet(pet: Pet): Pet

    @DELETE("/pet/{petId}")
    fun deletePet(@PathParam("petId") petId: Long)
}
```

### Tool Package (`*.tool`)

Contains CDI beans with LangChain4j `@Tool` annotations for AI/LLM integration.

**Features:**
- `@ApplicationScoped` for dependency injection
- `@RestClient` injection of the REST client
- `@Tool` annotations with descriptions
- Method names matching API operations
- Automatic wrapping of REST client calls

**Example:**

```kotlin
package io.swagger.petstore.tool

import dev.langchain4j.agent.tool.Tool
import jakarta.enterprise.context.ApplicationScoped
import org.eclipse.microprofile.rest.client.inject.RestClient
import io.swagger.petstore.client.PetstoreClient
import io.swagger.petstore.domain.Pet

@ApplicationScoped
class PetstoreTools {

    @RestClient
    lateinit var client: PetstoreClient

    @Tool("Finds Pets by status")
    fun findPetsByStatus(status: List<String>): List<Pet> {
        return client.findPetsByStatus(status)
    }

    @Tool("Find pet by ID")
    fun getPetById(petId: Long): Pet {
        return client.getPetById(petId)
    }
}
```

## Build Configuration

### `build.gradle.kts`

The generated Gradle build file includes:

```kotlin
plugins {
    kotlin("jvm") version "2.1.0"
    id("io.quarkus") version "3.15.3"
}

dependencies {
    implementation(enforcedPlatform("io.quarkus:quarkus-bom:3.15.3"))
    implementation("io.quarkus:quarkus-rest-client-jackson")
    implementation("io.quarkus:quarkus-resteasy-reactive-jackson")
    implementation("com.fasterxml.jackson.module:jackson-module-kotlin")
    implementation("dev.langchain4j:langchain4j-quarkus:0.36.2")

    testImplementation("io.quarkus:quarkus-junit5")
    testImplementation("io.rest-assured:rest-assured")
}

// Task for building domain+client only JAR
tasks.register<Jar>("domainClientJar") {
    dependsOn("compileKotlin")
    archiveClassifier.set("domain-client")
    from(sourceSets.main.get().output) {
        exclude("**/tool/**")
    }
}
```

### `settings.gradle.kts`

```kotlin
rootProject.name = "generated-sdk"
```

```properties
quarkus.application.name=Generated SDK
quarkus.http.port=8080

# Rest client configuration (replace with actual values)
petstore-api/mp-rest/url=https://petstore.swagger.io/v2
petstore-api/mp-rest/scope=javax.enterprise.context.ApplicationScoped
```

## Using the Generated Code

### Injecting the REST Client

```kotlin
import org.eclipse.microprofile.rest.client.inject.RestClient

@Path("/api")
class MyResource {

    @RestClient
    lateinit var petstoreClient: PetstoreClient

    @GET("/pets")
    fun getPets(): List<Pet> {
        return petstoreClient.findPetsByStatus(listOf("available"))
    }
}
```

### Using MCP Tools with LangChain4j

```kotlin
import dev.langchain4j.service AiServices
import dev.langchain4j.model.chat.ChatLanguageModel

val tools = PetstoreTools()
val aiService = AiServices.builder(Assistant::class.java)
    .chatLanguageModel(chatModel)
    .tools(tools)
    .build()
```
